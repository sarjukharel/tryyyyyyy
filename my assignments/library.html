Virtual Library Database Report

Objective

To understand the fundamentals of databases and their real-world applications by constructing a functional virtual library using CSV files and spreadsheet-based relationships.

Key Skills Demonstrated

  • Spreadsheet creation and manipulation
  • Data structure design
  • Establishing relationships using primary keys (PK) and foreign keys (FK)

Initial Task

Create and validate CSV files with structured data for a virtual library system. The data must include:

  • A books table with unique identifiers.
  • An authors table with more than two columns.
  • A transactions table with more than four columns.

Data Relationships (Foreign Keys)

  • book_ID in Transactions Table references book_ID in Main Library Table.
  • author in Main Library Table references author in Author Table.

Identified Issues & Improvements

  • Original data lacked normalization. Data was restructured into relational tables.
  • Primary and foreign keys ensure data integrity.
  • last_checkout_date shows as '#######', likely a spreadsheet formatting issue.
  • Recommended to add a Borrower Table for detailed user tracking.
